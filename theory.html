Page Replacement Theory

Short, viva-friendly explanation of why page replacement is needed and how FIFO, LRU and Optimal work, connected to the visualizer page for live demonstration.

What is Page Replacement?

In demand paging, processes use pages that are loaded into main memory frames on demand. When memory is full and a new page is needed, the OS must decide which existing page to remove. This decision is taken by a page replacement algorithm.

Goal: minimize page faults so that CPU does not waste time waiting for pages from disk. Our project compares three classic strategies used in Operating Systems.

1. FIFO (First In First Out)
  • Maintains pages in the order they entered memory.
  • When replacement is needed, it removes the oldest page.
  • Implementation is easy using a simple queue.
Idea: when a new page comes and frames are full,
  evict the page that has been in memory for the longest time.

Drawback: may lead to Bélády's anomaly, where increasing the number of frames actually increases page faults. You can test such strings in the visualizer.

2. LRU (Least Recently Used)
  • Assumes pages used recently are likely to be used again soon.
  • Replaces the page that has not been used for the longest time in the past.
  • Approximates the Optimal algorithm using past information.
Maintain last-used time for each frame.
On replacement, kick out the frame with the oldest last-used time.

LRU generally performs better than FIFO but needs extra data structures to track usage (counters, stacks, or hardware support like reference bits).

3. Optimal (MIN) Algorithm
  • Assumes we know the future reference string completely.
  • When replacement is needed, it removes the page whose next use is farthest in the future, or never used again.
  • Gives the theoretical minimum page faults for a given reference string.
For each page in a frame, look ahead in the reference string
and choose the one whose next occurrence is farthest (or doesn't appear).

Not implementable in real OS because we don't know the future, but it is extremely useful to evaluate other algorithms. Our visualizer uses Optimal as a baseline.
